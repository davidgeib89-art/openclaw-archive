# OM 3-6-9 Gate Flow (2026-02-17)

Status: Active execution doctrine  
Owner: Om team  
Purpose: Enforce staged quality progression with fast diagnosis and full-system truth checks

## 1) Why 3-6-9

The flow balances speed and rigor:
1. Gate 3 catches regressions early.
2. Gate 6 checks broader stability before expensive full sweeps.
3. Gate 9 remains the final truth check for promotion claims.

This keeps single-variable discipline while protecting the full battery standard.

3. If score delta is negative vs lock, continue without promotion.

### Gate 6 (Stability)

### Gate 9 (Release truth)
Rituals:
1. Full 9-ritual battery per `OM_PROTO33_RITUAL_TEST_BATTERY_2026-02-16.md`

Pass rule:
1. >= 7/9 passes
2. >= 4 strong passes
3. Zero unauthorized side-effect writes
4. Zero loop cascades

## 3) Promotion policy

1. R060 stays lock until a Gate 9 cycle is passing and non-regressive.
2. Gate 3 or Gate 6 pass alone does not promote lock.
3. Any hard-gate regression resets progression to targeted remediation.

## 4) Execution sequence per change batch

1. Apply one variable only.
2. Run Gate 3.
3. If Gate 3 fails: remediate immediately.
4. If Gate 3 passes: run Gate 6.
5. If Gate 6 passes: run Gate 9.
6. Promote only after Gate 9 confirmation.
