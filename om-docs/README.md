# Om Documentation Hub

This directory is the canonical documentation root for Project Om inside OpenClaw.

## Canonical entrypoints

- Current source of truth: `om-docs/current/OM_CANON.md`
- Current architecture: `om-docs/current/OM_ARCHITECTURE.md`
- Current roadmap: `om-docs/current/OM_ROADMAP.md`

## Legacy material

Historical files, handovers, reports, and temporary planning notes are preserved in:

- `om-docs/archive/`
- `om-docs/archive/root-legacy/2026-03-16/`

Nothing was deleted. Older documents were moved to reduce ambiguity for future AI runs and for humans onboarding with zero prior context.

## How to use this docs tree

1. Read `om-docs/current/OM_CANON.md` first.
2. Read `om-docs/current/OM_ARCHITECTURE.md` for implementation-level orientation.
3. Use `om-docs/current/OM_ROADMAP.md` for active and upcoming work.
4. Use archive files only as historical context, not as operational source of truth.
